# QA Matrix: Lessons & Reports

Ниже — ручная матрица для боевой проверки модуля учета занятий и отчетов.

## Как проходить
- `[x]` — проверено агентом (авто-прогон/проверка кода).
- `[ ]` — нужен ручной UI-прогон на устройстве (визуал/навигация/UX).
- Если пункт упал — фиксируй шаг, фактический результат, скрин/видео.

## Подготовка
- Пользователь 1: обычный преподаватель (private access).
- Пользователь 2: суперпользователь (private access + superuser).
- Есть минимум 3 ученика, у 1-го есть история занятий, у 2-го пустая история.
- Приложение обновлено до актуального билда, сервер перезапущен.

## Фактический прогон агентом (сделано)
- `dart analyze .` — OK
- `flutter test` — OK
- `flutter build apk --debug` — OK
- `npm run smoke:accounting` — OK
- `npm run smoke:accounting:edge` — OK
- Проверка измененной логики по коду (UI/API связки, fallback, guards) — выполнена
