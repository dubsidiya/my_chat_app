## 1. Что такое программа и код

**Программа** — это набор инструкций для компьютера. Компьютер выполняет их по порядку (или переходит к разным веткам в зависимости от условий). **Код** — эти инструкции, записанные в виде текста на **языке программирования**. У языка есть правила написания (синтаксис): где ставить скобки, точки с запятой, как называть переменные.

**Dart** — один из таких языков. На нём пишут приложения Flutter (в том числе наш чат). Код хранится в **файлах** с расширением `.dart`. Один файл может содержать одну логическую часть программы (например, описание модели «пользователь» или экран входа).

---

## 2. Первая программа на Dart

Минимальная программа на Dart — функция **main**. С неё начинается выполнение.

```dart
void main() {
  print('Привет, мир!');
}
```

- `void` — функция ничего не возвращает.
- `main` — имя функции, точка входа.
- `()` — список аргументов (здесь пустой).
- `{ ... }` — тело функции.
- `print('...')` — вывести текст в консоль.

Запуск: сохраните код в файл `hello.dart` и выполните в терминале `dart run hello.dart` (если установлен Dart SDK). В проекте Flutter точка входа тоже `main()` в `lib/main.dart`, но там внутри вызывается `runApp(...)` для запуска интерфейса.

## 3. Переменные и типы

**Переменная** — именованное место для значения. Мы даём имя (например, `userName`) и сохраняем туда данные, чтобы потом использовать их в коде.

**Тип** говорит, что за данные хранятся: целое число, дробное, строка текста, «да/нет». В Dart можно указывать тип явно или позволить компилятору вывести его самому.

Основные типы:

| Тип     | Что хранит        | Пример значения   |
|---------|-------------------|--------------------|
| `int`   | Целое число       | `42`, `0`, `-5`    |
| `double`| Дробное число     | `3.14`, `0.5`      |
| `String`| Строка текста     | `'Привет'`, `"a"`  |
| `bool`  | Истина/ложь       | `true`, `false`    |

Примеры:

```dart
void main() {
  int age = 25;
  String name = 'Анна';
  bool isActive = true;
  double price = 99.99;

  print(name);   // Анна
  print(age + 1); // 26
}
```

Ключевое слово **final** означает: переменная задаётся один раз и потом не меняется. Это помогает избежать случайных изменений.

```dart
final String id = 'user-123';
// id = 'другое';  // так сделать уже нельзя — ошибка
```

В проекте чата так описаны, например, поля модели пользователя: `final String id;` и `final String email;` — у конкретного пользователя они не меняются после создания объекта.

## 4. Условия: if и else

Часто нужно выполнить разный код в зависимости от условия: **если** выполнено — сделай одно, **иначе** — другое.

Синтаксис:

```dart
if (условие) {
  // код, если условие истинно
} else {
  // код, если условие ложно
} 
```

Условие — это выражение типа `bool` или сравнение (==, !=, <, >, <=, >=).

Пример:

```dart
void main() {
  int score = 85;
  if (score >= 60) {
    print('Сдал');
  } else {
    print('Не сдал');
  }
}
```

Можно проверять несколько вариантов через **else if**:

```dart
if (score >= 90) {
  print('Отлично');
} else if (score >= 70) {
  print('Хорошо');
} else if (score >= 50) {
  print('Удовлетворительно');
} else {
  print('Не сдал');
}
```

В приложении чата так обрабатывают ответ сервера: если код ответа 200 — разбираем данные, если 401 — «неверный логин или пароль», иначе — «ошибка сервера».

## 5. Циклы: for и while

**Цикл** — повторение одного и того же кода несколько раз.

**for** — когда известно, сколько раз повторять или по какому списку проходить:

```dart
for (int i = 0; i < 5; i++) {
  print('Шаг $i');
}
// Шаг 0, Шаг 1, ... Шаг 4

// перебор элементов списка
List<String> names = ['Анна', 'Борис', 'Вера'];
for (String name in names) {
  print('Привет, $name!');
}
```

**while** — повторять, пока условие истинно:

```dart
int n = 0;
while (n < 3) {
  print(n);
  n++;
}
```

В проекте цикл `for` используется, например, при разборе списка чатов из JSON: для каждого элемента массива создаётся объект `Chat` и добавляется в список.

## 6. Функции

**Функция** — именованный блок кода с аргументами и (опционально) возвращаемым значением. Вызываем по имени — получаем результат и не дублируем один и тот же код в разных местах.

Объявление:

```dart
тип_возврата имя_функции(тип1 аргумент1, тип2 аргумент2) {
  // тело
  return значение;
}
```

Примеры:

```dart
int sum(int a, int b) {
  return a + b;
}

String greet(String name) {
  return 'Привет, $name!';
}

void printMessage(String msg) {
  print(msg);
  // return не нужен — void
}
```

Вызов: `sum(2, 3)` вернёт `5`; `greet('Мир')` — строку `'Привет, Мир!'`.

В приложении чата функции используются повсеместно: от проверки логина и пароля до форматирования даты последнего сообщения в списке чатов.

## 7. Списки (List)

**List** — упорядоченный список элементов. Элементы могут быть одного типа или общего (dynamic).

Создание и доступ по индексу (индексы с нуля):

```dart
List<String> fruits = ['яблоко', 'груша', 'слива'];
print(fruits[0]);  // яблоко
print(fruits.length);  // 3

fruits.add('вишня');
fruits.removeAt(1);  // удалить элемент с индексом 1
```

Пустой список с указанием типа:

```dart
List<int> numbers = [];
numbers.add(10);
numbers.add(20);
```

Перебор:

```dart
for (var item in fruits) {
  print(item);
}
```

В проекте так хранятся, например, список чатов (`List<Chat>`) и список сообщений (`List<Message>`).

## 8. Словари (Map)

**Map** — пары «ключ → значение». По ключу получаем значение. Ключи часто — строки, значения могут быть разного типа (поэтому часто используют `Map<String, dynamic>`).

```dart
Map<String, dynamic> user = {
  'id': '1',
  'name': 'Вася',
  'age': 30,
};

print(user['name']);  // Вася
user['email'] = 'vasya@mail.ru';  // добавить или изменить
```

Проверка наличия ключа:

```dart
if (user.containsKey('name')) {
  print(user['name']);
}
```

Ответ сервера в формате JSON после разбора обычно оказывается именно `Map<String, dynamic>` (объект) или `List<dynamic>` (массив).

## 9. Парсинг JSON пошагово

JSON — текстовый формат данных. Объект в фигурных скобках `{ "ключ": "значение" }`, массив в квадратных `[ ... ]`. Им обмениваются клиент и сервер.

### 9.1. Что такое JSON

Пример одного объекта:

```json
{
  "id": "123",
  "email": "user@example.com",
  "active": true
}
```

Пример массива объектов:

```json
[
  { "id": "1", "title": "Первый" },
  { "id": "2", "title": "Второй" }
]
```

В Dart такой текст разбирается функцией **jsonDecode** из библиотеки **dart:convert**. Она возвращает:
- объект JSON → `Map<String, dynamic>`;
- массив JSON → `List<dynamic>`.

### 9.2. Парсинг из строки

```dart
import 'dart:convert';

void main() {
  String jsonString = '{"id": "1", "name": "Тест"}';
  Map<String, dynamic> data = jsonDecode(jsonString);
  print(data['name']);  // Тест
  print(data['id']);   // 1
}
```

Если строка содержит массив:

```dart
String listJson = '[1, 2, 3]';
List<dynamic> list = jsonDecode(listJson);
print(list[0]);  // 1
```

### 9.3. JSON из ответа сервера (HTTP)

В приложении JSON чаще всего приходит в теле HTTP-ответа. У объекта ответа есть поле **body** — это строка. Её и передаём в **jsonDecode**:

```dart
import 'dart:convert';
import 'package:http/http.dart' as http;

Future<void> fetchUser() async {
  final response = await http.get(Uri.parse('https://api.example.com/user'));
  if (response.statusCode == 200) {
    final data = jsonDecode(response.body);
    String name = data['name'];
    int id = data['id'];
    print('$name, id: $id');
  }
}
```

Важно: тип у `data['id']` может быть и int, и String (в зависимости от сервера). В коде часто приводят к нужному типу: `data['id'].toString()` или `int.tryParse(data['id'].toString())`.

### 9.4. JSON из файла

**Вариант 1: файл в папке assets (Flutter).** Сначала читаем файл в строку, потом парсим:

```dart
import 'dart:convert';
import 'package:flutter/services.dart' show rootBundle;

Future<Map<String, dynamic>> loadJsonFromAssets() async {
  final String raw = await rootBundle.loadString('assets/config.json');
  return jsonDecode(raw) as Map<String, dynamic>;
}
```

Файл должен быть указан в `pubspec.yaml` в секции `flutter: assets:`.

**Вариант 2: файл по пути на диске (Dart или Flutter с path_provider):**

```dart
import 'dart:convert';
import 'dart:io';

Future<Map<String, dynamic>> loadJsonFromFile(String path) async {
  final file = File(path);
  final String raw = await file.readAsString();
  return jsonDecode(raw) as Map<String, dynamic>;
}
```

### 9.5. От Map к своему классу (fromJson)

Чтобы не работать с «голыми» ключами и `dynamic`, данные оборачивают в **модель** — класс с полями. В нём делают фабричный конструктор **fromJson**, который принимает `Map<String, dynamic>` и возвращает объект класса.

Правила:
- Проверять наличие ключа или использовать `json['key'] ?? значение_по_умолчанию`.
- Приводить типы: числа и булевы значения с сервера могут приходить в разном виде (int, double, String "true"), поэтому в проекте часто есть вспомогательные функции вроде `_parseBool`, `_parseInt`.

Пример модели пользователя:

```dart
class User {
  final String id;
  final String email;
  final String name;

  User({required this.id, required this.email, required this.name});

  factory User.fromJson(Map<String, dynamic> json) {
    return User(
      id: (json['id'] ?? '').toString(),
      email: (json['email'] ?? '').toString(),
      name: (json['name'] ?? '').toString(),
    );
  }
}
```

Использование:

```dart
final data = jsonDecode(response.body);
User user = User.fromJson(data);
print(user.name);
```

### 9.6. Парсинг списка JSON-объектов

Если сервер вернул массив объектов, после **jsonDecode** получаем **List<dynamic>**. Каждый элемент приводим к **Map** и передаём в **fromJson**:

```dart
List<User> parseUserList(String jsonString) {
  final List<dynamic> raw = jsonDecode(jsonString);
  return raw.map((e) => User.fromJson(e as Map<String, dynamic>)).toList();
}
```

Или в цикле:

```dart
List<User> users = [];
for (var item in jsonDecode(jsonString) as List) {
  users.add(User.fromJson(item as Map<String, dynamic>));
}
```

В проекте чата так разбирают список чатов в `ChatsService` и список сообщений в `MessagesService`.

### 9.7. Обратная операция: объект → JSON (toJson)

Чтобы отправить данные на сервер или сохранить в файл, модель превращают в Map, затем в строку:

```dart
Map<String, dynamic> toJson() => {
  'id': id,
  'email': email,
  'name': name,
};

String jsonString = jsonEncode(user.toJson());
```

**jsonEncode** из `dart:convert` превращает Map или List в строку JSON.

### 9.8. Типичные ошибки при парсинге JSON

- **Ключа нет в ответе** — использовать `json['key'] ?? default`, иначе возможен null или исключение в зависимости от типа поля.
- **Тип значения не совпадает**: сервер прислал число, а вы ожидаете строку — явно приводить: `.toString()`, `int.tryParse(...)`.
- **Вложенный объект**: например, `json['address']` — тоже Map; для него пишут отдельную модель и вызывают `Address.fromJson(json['address'])`.
- **Ошибка в формате JSON** — `jsonDecode` выбросит исключение; оборачивать вызов в try/catch и показывать пользователю сообщение об ошибке или логировать.

## 10. Классы и объекты

**Класс** — описание «чертежа»: какие поля и методы у сущности. **Объект** — конкретный экземпляр класса с конкретными значениями полей.

Пример:

```dart
class Chat {
  final String id;
  final String name;
  final int unreadCount;

  Chat({required this.id, required this.name, this.unreadCount = 0});
}

// использование
Chat c = Chat(id: '1', name: 'Чат с Машей', unreadCount: 2);
print(c.name);
```

В проекте модели **Chat**, **Message**, **User** — классы с полями и фабричным методом **fromJson** для создания объекта из JSON.

### 10.1. Перечисления (enum)

Когда у величины может быть только несколько фиксированных вариантов (статус сообщения: отправлено / доставлено / прочитано), удобно использовать **enum** — перечисление. Имена значений пишут с маленькой буквы.

```dart
enum MessageStatus { sending, sent, delivered, read }

// использование
MessageStatus status = MessageStatus.sent;
if (status == MessageStatus.read) { ... }
```

В **switch** по enum компилятор может потребовать обработать все варианты. В проекте в **lib/models/message.dart** объявлен `enum MessageStatus` — его используют при отображении статуса сообщения в чате.

## 11. Null-safety: ? и ??

В Dart переменная по умолчанию не может иметь значение **null** (отсутствие значения), если тип не помечен как nullable. Это уменьшает количество ошибок «null pointer».

- **Тип с `?`** — переменная может быть null: `String? name;`
- **`??`** — если слева null, подставить значение справа: `json['name'] ?? 'Без имени'`
- **`!`** — утверждение «здесь точно не null» (если окажется null — ошибка при выполнении)

Пример:

```dart
String? nickname;
print(nickname ?? 'гость');  // гость
nickname = 'Вася';
print(nickname ?? 'гость');  // Вася
```

## 12. Обработка ошибок: try и catch

Опасный код (сеть, разбор JSON, работа с файлами) оборачивают в **try**. Если внутри произошло исключение, выполнение переходит в **catch** — там можно показать сообщение пользователю или залогировать ошибку.

```dart
try {
  final data = jsonDecode(response.body);
  return User.fromJson(data);
} catch (e) {
  print('Ошибка парсинга: $e');
  rethrow;  // или return null; / throw Exception('Понятное сообщение');
}
```

В приложении чата так обрабатывают ошибки входа, загрузки чатов и сообщений: в catch обновляют состояние экрана (текст ошибки, кнопка «Повторить») и при необходимости пишут в консоль в режиме отладки.
