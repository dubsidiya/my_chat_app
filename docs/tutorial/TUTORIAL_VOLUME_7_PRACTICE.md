## 1. Задачи по Dart и JSON

- **Задача 1.1.** Напиши функцию `parseProductList(String jsonString)`, которая принимает JSON-массив объектов с полями `id`, `name`, `price` и возвращает `List<Product>`. Класс Product с fromJson опиши сам.
- **Задача 1.2.** Добавь в модель Product метод `toJson()`. Напиши тест: создать Product, вызвать toJson, затем fromJson и проверить равенство полей.
- **Задача 1.3.** В JSON приходит поле `is_active` (может быть bool, int 0/1, строка "true"/"false"). Напиши функцию `bool parseBool(dynamic value)` и используй её в fromJson.

## 2. Задачи по Flutter UI

- **Задача 2.1.** Экран с заголовком, полем ввода и кнопкой «Сохранить». По нажатию — сохранить текст в переменную состояния и показать под кнопкой «Сохранено: <текст>».
- **Задача 2.2.** Список из 10 элементов (например, строки «Элемент 1» … «Элемент 10») через ListView.builder; при нажатии на элемент — SnackBar с текстом элемента.
- **Задача 2.3.** Переиспользуемый виджет Card с title и subtitle; используй его в списке из трёх карточек с разными данными.

---

## 3. Задачи по навигации и диалогам

- **Задача 3.1.** Экран A с кнопкой «Выбрать». По нажатию — открыть экран B с двумя кнопками «Вариант 1» и «Вариант 2». По нажатию на вариант — pop с результатом и на экране A показать «Выбрано: Вариант N».
- **Задача 3.2.** Диалог подтверждения с текстом «Удалить элемент?» и кнопками «Отмена» / «Удалить». Возвращать true/false. На вызывающем экране при true вывести SnackBar «Удалено».
- **Задача 3.3.** Bottom sheet с тремя пунктами меню; по выбору — закрыть sheet и выполнить действие (например, показать SnackBar с названием пункта).

---

## 4. Задачи по сети и API

- **Задача 4.1.** Напиши метод `Future<User?> fetchUser(String id)` в стиле проекта: GET запрос к `$baseUrl/users/$id`, заголовки из _getAuthHeaders(), разбор JSON в User. Обработай 404 и сетевые ошибки.
- **Задача 4.2.** Экран с кнопкой «Загрузить». По нажатию — запрос к API (например, список чатов), показ индикатора загрузки, при успехе — список, при ошибке — кнопка «Повторить».
- **Задача 4.3.** Сохранить токен после логина в StorageService (или в переменную для теста) и при следующем запросе подставлять в Authorization. Проверь на одном запросе (например, GET /chats).

---

## 5. Сборка и запуск

- **Задача 5.1.** В корне проекта выполни `flutter pub get`, затем `flutter run`. Запусти на доступном устройстве или эмуляторе (Chrome для web).
- **Задача 5.2.** Собери релизную версию для web: `flutter build web`. Где лежит выход (build/web)? Открой index.html в браузере локально.
- **Задача 5.3.** Запусти с переопределённым API: `flutter run --dart-define=API_BASE_URL=https://твой-сервер`. Убедись, что запросы уходят на новый URL (по логам или поведению).

## 6. Мини-проекты

- **Мини-проект 1.** «Список заметок»: один экран, список заголовков заметок в состоянии, кнопка «Добавить» открывает диалог с полем ввода; по сохранению — добавить в список и закрыть диалог. Без сохранения на диск.
- **Мини-проект 2.** «Просмотр пользователя»: экран с полем ввода id и кнопкой «Загрузить». По нажатию — GET запрос к условному API (или мок), отображение имени и email в карточке. Обработка ошибок и загрузки.
- **Мини-проект 3.** В существующем my_chat_app: добавить на экран профиля переключатель «Тёмная/светлая тема» и сохранять выбор в SharedPreferences; при запуске применять сохранённую тему (потребуется доработка main.dart и темы).
