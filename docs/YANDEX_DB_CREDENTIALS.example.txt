# Пример учётных данных кластера БД (НЕ КОММИТИТЬ — файл с реальными данными в .gitignore)
# Использование: скопируй в YANDEX_DB_CREDENTIALS.txt и подставь реальные значения из консоли Yandex Cloud.
# Файл YANDEX_DB_CREDENTIALS.txt никогда не должен попадать в репозиторий.

Хост:     YOUR_CLUSTER.mdb.yandexcloud.net
Порт:     6432 (Connection Manager) или 5432 (прямое)
БД:       chat_db
Пользователь: chat_app
Пароль:   REDACTED

Строка подключения (подставь в DATABASE_URL на Render или в .env на ВМ):
postgresql://chat_app:REDACTED@YOUR_CLUSTER.mdb.yandexcloud.net:6432/chat_db?sslmode=require

Важно:
1. В консоли Yandex открой порт 6432 (и при необходимости 5432) в группах безопасности кластера для твоего IP или 0.0.0.0/0.
2. Реальный файл YANDEX_DB_CREDENTIALS.txt храни только локально; после утечки — смени пароль БД и удали файл из истории git (см. docs/SECRETS_ROTATION.md).
