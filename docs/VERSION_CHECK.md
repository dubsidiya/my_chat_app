# Проверка версии приложения

Функция «есть ли у пользователя последняя версия» включена. При устаревшей версии показывается диалог и (на **веб**) — баннер с инструкцией, как обновиться.

## Веб (Vercel): сброс кэша

На веб-версии браузер может отдавать старую сборку из кэша. Поэтому при доступном обновлении пользователь видит:

- **Диалог** при входе: текст с инструкцией и кнопка «Обновить страницу» (перезагрузка).
- **Баннер** сверху экрана: «Доступна новая версия. Обновите страницу: Ctrl+Shift+R (Win/Linux) или Cmd+Shift+R (Mac).» и кнопка «Обновить».

Рекомендуется после каждого деплоя на Vercel обновлять на бэкенде переменную **APP_LATEST_VERSION** до текущей версии из `pubspec.yaml` (например `1.0.1`), чтобы клиенты с закэшированной старой версией получили подсветку обновления.

## Что есть в коде

- **Бэкенд:** `GET /version` — отдаёт `minVersion`, `latestVersion`, `forceUpdate`, `message`, ссылки на магазины. Параметры задаются через переменные окружения (см. `my_serve_chat_test/.env.example`, блок «Проверка версии приложения»).
- **Flutter:** сервис `lib/services/version_check_service.dart` — запрашивает `/version`, сравнивает с текущей версией из `package_info_plus`. На **web** показывает инструкцию по обновлению страницы и кнопку перезагрузки; на iOS/Android — кнопки в магазин.
- **Вызов:** в `lib/screens/main_tabs_screen.dart` при входе вызываются `VersionCheckService.check()` и `showDialogIfNeeded()`; при наличии обновления на web показывается баннер сверху.

## Как настроить после публикации в сторе

1. На **сервере** в `.env` задать (и обновлять при каждом релизе веба/приложений):

   ```env
   APP_MIN_VERSION=1.0.0
   APP_LATEST_VERSION=1.0.0
   APP_STORE_URL_ANDROID=https://play.google.com/store/apps/details?id=ВАШ_PACKAGE
   APP_STORE_URL_IOS=https://apps.apple.com/app/idXXXXX
   ```

   При выходе новой версии — обновить `APP_LATEST_VERSION` (и при необходимости `APP_MIN_VERSION`) и перезапустить сервер. Для веба после каждого деплоя на Vercel поднимать `APP_LATEST_VERSION` до версии из `pubspec.yaml`.

2. В приложении версия берётся из `pubspec.yaml` (поле `version`). Перед сборкой веба или мобильных релизов нужно увеличить версию там.

После этого при каждом входе пользователя будет запрос к `/version` и при устаревшей версии — диалог и (на веб) баннер с предложением обновиться.
