### Для кого он

Для вас, если вы:
- никогда не писали код и не знаете, что такое «переменная» или «функция»;
- хотите понять, как устроены приложения на телефоне или в браузере;

Мы не предполагаем, что вы уже что-то устанавливали или программировали. Всё необходимое будет введено по ходу.

### Что мы будем изучать

1. **Сначала — общие идеи.** Что такое программа и код, зачем нужны файлы и папки, как приложение на телефоне связано с «компьютером в интернете» (сервером).
2. **Потом — основы языка Dart.** Это язык, на котором написан наш чат. Мы разберём: как запомнить данные (переменные), как принять решение «если одно — сделай так, иначе — иначе» (условия), как повторять действия (циклы), как вынести повторяющиеся действия в «рецепт» (функции) и как работать со списками и структурами данных.

4. **В конце — проект целиком.** Как при запуске приложения выбирается первый экран, кто за что отвечает (сервисы, модели), как данные приходят с сервера и превращаются в то, что вы видите на экране.

После этого вы сможете открыть любой файл в `lib/`, понять его роль и, например, изменить надпись на кнопке или добавить простое поле на экран.

### Что нужно от вас

- Компьютер (Windows, macOS или Linux) или возможность работать на нём.
- Время и желание читать по порядку. Установку Flutter и запуск приложения мы разберём в [Части E](#часть-e-дополнительно) — к этому моменту вы уже будете понимать, что именно запускается.

## Часть A. Введение в программирование

В этой части — самые базовые вещи: что такое программа и код, как приложение связано с интернетом и зачем в проекте столько папок и файлов.

### Что вы видите, когда открываете приложение

Когда вы открываете чат на телефоне, вы видите экран: кнопки, поля ввода, список переписок. Всё это нарисовано по **инструкциям**, записанным в виде текста в файлах. Этот текст и есть **код**. Программист пишет код; специальная программа (компилятор) переводит его в форму, которую процессор умеет выполнять; в результате на экране появляются нужные кнопки и реакции на нажатия. То есть: **то, что вы видите в приложении — результат выполнения кода.**

### Что такое программа, код и компиляция

**Простыми словами:**

- **Программа** — набор инструкций для компьютера. Как рецепт: «возьми яйцо, взбей, добавь муку». Компьютер выполняет инструкции по порядку (или переходит к разным шагам в зависимости от условий — об этом в части B).
- **Код** — этот набор инструкций, записанный текстом в **файлах**. Код пишут на **языке программирования** — у него свои правила (синтаксис). В нашем проекте язык — **Dart**.
- **Файл** — один «документ» на диске: один кусок кода. **Папка** — группа файлов, чтобы было понятно, что где лежит (все экраны — в `screens`, работа с сетью — в `services`).
- **Компиляция** — перевод кода из текста в форму, которую процессор умеет выполнять. В итоге из ваших файлов получается работающее приложение.

**Теория:** исходный код (файлы `.dart`) обрабатывается компилятором Dart. Flutter собирает приложение под разные платформы (iOS, Android, Web) — один и тот же код переводится в нужный формат для каждой.

### Клиент и сервер на примере чата

Представьте обычный чат: вы входите по логину и паролю, видите список чатов, открываете переписку, пишете сообщение. Где хранятся пароли, чаты и сообщения? Не только у вас в телефоне — иначе другой человек не увидел бы ваше сообщение. Данные хранятся на **сервере** — это программа, которая работает на компьютере в интернете и постоянно «ждёт» запросы.

**Простыми словами:**

- **Клиент** — то, с чем вы взаимодействуете: приложение на телефоне или в браузере. Оно рисует экраны, кнопки, поля ввода и **отправляет запросы** на сервер: «войти с таким-то логином», «дать список моих чатов», «отправить это сообщение».
- **Сервер** — программа в интернете. Она проверяет логин и пароль, отдаёт список чатов, сохраняет сообщения и рассылает их другим участникам. Она не рисует картинку на вашем экране — она только обрабатывает запросы и отправляет ответы (обычно в виде данных: список чатов, текст сообщения и т.д.).

Когда вы нажимаете «Войти» или «Отправить», приложение (клиент) отправляет запрос на сервер; сервер обрабатывает его и возвращает ответ (например, «вход выполнен» или список чатов). Так устроен любой чат, почта, соцсети.

**Теория:** это называется архитектурой «клиент — сервер». Обмен идёт по протоколам: **HTTP** — отдельные запросы и ответы (загрузить чаты, отправить сообщение); **WebSocket** — постоянное соединение, чтобы новые сообщения приходили мгновенно, без перезагрузки. **API** (Application Programming Interface) — набор правил: какие «адреса» вызывать, в каком формате передавать данные (в нашем проекте используется текст в формате JSON).

### Файлы и папки проекта

Код приложения разложен по папкам не случайно. Так проще искать нужное: «всё про экраны — в одной папке, всё про работу с сетью — в другой». Ниже — что за что отвечает в нашем проекте. Пока можно просто пробежать глазами; по мере изучения частей B и C вы будете открывать эти файлы и узнавать их по содержимому.

## Часть B. Dart с нуля

В этой части мы разбираем **язык Dart** — на нём написан весь код нашего чата. Мы не будем грузить вас теорией «на будущее»: каждое понятие нужно нам, чтобы потом спокойно читать файлы из `lib/`. Идём от простого к более сложному: сначала как «запомнить» данные и как принять решение в коде, потом как вынести повторяющиеся действия в «рецепты» (функции) и как работать со списками и структурами данных.

### Зачем нужны переменные и типы

В жизни мы постоянно что-то запоминаем: «имя пользователя — Вася», «количество непрочитанных сообщений — 5». В коде то же самое: программе нужно **где-то хранить** эти данные и **обращаться к ним по имени**. Такое «место» с именем называется **переменная**. Например: переменная `userName` хранит имя пользователя, переменная `unreadCount` — число непрочитанных.

Чтобы компьютер правильно обрабатывал данные (числа — складывать, текст — показывать, да/нет — использовать в условиях), нужно указать **тип** переменной: число целое (`int`), число с дробной частью (`double`), строка текста (`String`), «да или нет» (`bool`). Тогда и вы, и компилятор понимаете, что в этой переменной лежит.

### Переменные и типы

**Простыми словами:** переменная — это именованное «место» для значения. Тип говорит, что за значение там хранится: число, текст, да/нет и т.д.

В Dart основные типы: `int` (целые числа), `double` (дробные), `String` (строка текста), `bool` (логическое значение: true/false). Тип можно писать явно или позволить компилятору вывести его по значению.

**Пример из проекта** — поля модели `User` в `lib/models/user.dart`:

```dart
class User {
  final String id;
  final String email;

  User({required this.id, required this.email});
}
```

Здесь `id` и `email` — переменные (поля класса) типа `String`. Ключевое слово `final` означает, что после присвоения значение менять нельзя.

**Ещё пример** — форматирование времени в `lib/screens/home_screen.dart`:

```dart
String _formatLastMessageTime(String? iso) {
  if (iso == null || iso.isEmpty) return '';
  try {
    final dt = DateTime.parse(iso).toLocal();
    final now = DateTime.now();
    final sameDay = dt.year == now.year && dt.month == now.month && dt.day == now.day;
    if (sameDay) {
      return DateFormat('HH:mm').format(dt);
    }
    return DateFormat('dd.MM').format(dt);
  } catch (_) {
    return '';
  }
}
```

Здесь `dt`, `now`, `sameDay` — локальные переменные; тип выводится автоматически (`DateTime`, `bool`).

### Условия и циклы

В жизни мы постоянно делаем выбор: **если** холодно — надень куртку, **иначе** — футболку. И повторяем одно и то же: **для каждого** человека в комнате — поздоровайся. В коде то же самое: **условие** — выполнить один кусок кода или другой в зависимости от того, истинно ли выражение; **цикл** — выполнить кусок кода много раз (например, для каждого чата в списке).

Синтаксис условия в Dart: `if (условие) { ... } else { ... }`.

**Пример из проекта** — проверка ответа сервера в `lib/services/auth_service.dart`:

```dart
if (response.statusCode == 200) {
  final data = jsonDecode(response.body);
  // сохраняем токен, данные пользователя...
  return data;
} else if (response.statusCode == 401) {
  throw Exception('Неверный логин или пароль');
} else {
  throw Exception('Ошибка подключения к серверу (${response.statusCode})');
}
```

**Циклы** — повторение действий. `for` — когда известно, сколько раз или по какому списку итерировать; `while` — пока условие истинно. В проекте цикл `for` встречается, например, при разборе списка чатов из JSON в `lib/services/chats_service.dart`:

```dart
for (var chatJson in data) {
  try {
    chats.add(Chat.fromJson(chatJson as Map<String, dynamic>));
  } catch (e) {
    // пропускаем проблемный чат
  }
}
```

### Функции

Когда одно и то же действие нужно выполнять в разных местах (например, «проверить логин и пароль» или «преобразовать дату в красивый вид»), его выносят в отдельный **рецепт** с именем — это и есть **функция**. У неё есть **аргументы** (то, что мы передаём: логин, пароль, дата) и **результат** (то, что функция возвращает: успех/ошибка, отформатированная строка). Вызвали функцию по имени — получили результат и продолжаете код.

**Примеры из проекта:**

- Функция без возвращаемого значения (void), с двумя параметрами — в `lib/services/auth_service.dart`:

```dart
Future<Map<String, dynamic>?> loginUser(String username, String password) async {
  try {
    final response = await http.post(
      Uri.parse('$baseUrl/auth/login'),
      headers: {'Content-Type': 'application/json'},
      body: jsonEncode({'username': username, 'password': password}),
    );
    // ...
  }
}
```

- Вспомогательная функция в модели `Chat` — `lib/models/chat.dart`:

```dart
static bool _parseBool(dynamic value) {
  if (value is bool) return value;
  if (value is int) return value == 1;
  if (value is String) {
    return value.toLowerCase() == 'true' || value == '1';
  }
  return false;
}
```

Здесь `static` означает, что функция принадлежит классу, а не экземпляру; её вызывают как `Chat._parseBool(...)`.

### Коллекции: List и Map

Часто нужно хранить не одно значение, а много: список чатов, список сообщений. Или пары «название — значение»: например, «логин» → «Вася», «пароль» → «секрет». В Dart для этого есть два основных типа:

- **List (список)** — упорядоченный перечень элементов. Как список покупок: первый пункт, второй, третий. В проекте так хранятся, например, список чатов (`List<Chat>`) и список сообщений (`List<Message>`).
- **Map (отображение, «словарь»)** — пары «ключ → значение». По ключу получаем значение: например, по ключу `'token'` получаем сохранённый токен. Ответ сервера часто приходит в виде такой структуры (в формате JSON).

В проекте списки чатов и сообщений хранятся в `List<Chat>` и `List<Message>`. Пример из `lib/screens/home_screen.dart`:

```dart
List<Chat> _chats = [];
// ...
final chats = await _chatsService.fetchChats(widget.userId);
setState(() {
  _chats = chats;
});
```

Данные с сервера часто приходят в виде `Map<String, dynamic>` (JSON-объект). Пример чтения из ответа в `lib/services/auth_service.dart`:

```dart
final data = jsonDecode(response.body);
if (data['token'] != null) {
  final userIdentifier = data['username'] ?? data['email'] ?? '';
  final displayName = data['displayName']?.toString();
  await StorageService.saveUserData(
    data['id'].toString(),
    userIdentifier,
    data['token'],
    displayName: displayName,
  );
}
```

Здесь `data` — это `Map`: ключи `'token'`, `'username'`, `'id'` и т.д.

### Парсинг JSON: пошагово

**Что такое JSON.** JSON (JavaScript Object Notation) — это текстовый формат данных. Им обмениваются клиент и сервер: и человек может прочитать, и программа легко разбирает. Выглядит так: объект — в фигурных скобках `{ "ключ": "значение" }`, список — в квадратных `[ 1, 2, 3 ]`. Пример одного объекта пользователя:

```json
{
  "id": "123",
  "email": "user@example.com",
  "name": "Вася"
}
```

Пример списка объектов (например, чатов):

```json
[
  { "id": "1", "name": "Чат с Машей" },
  { "id": "2", "name": "Рабочая группа" }
]
```

**Шаг 1: из строки в данные Dart.** В Dart для разбора JSON нужна библиотека `dart:convert`. Функция **`jsonDecode(строка)`** превращает строку в типы Dart: объект JSON → `Map<String, dynamic>`, массив JSON → `List<dynamic>`. Пример:

```dart
import 'dart:convert';

void main() {
  String jsonString = '{"id": "1", "name": "Тест"}';
  Map<String, dynamic> data = jsonDecode(jsonString);
  print(data['name']); // Тест
  print(data['id']);   // 1
}
```

**Шаг 2: JSON из ответа сервера.** В приложении JSON чаще всего приходит в теле HTTP-ответа. У объекта `response` есть поле `body` — это строка. Её и передаём в `jsonDecode`:

```dart
import 'dart:convert';
import 'package:http/http.dart' as http;

// после запроса:
final response = await http.get(Uri.parse(url));
if (response.statusCode == 200) {
  final data = jsonDecode(response.body);  // body — строка с JSON
  // data — Map или List в зависимости от того, что вернул сервер
  String token = data['token'];
  int id = data['id'];
}
```

**Шаг 3: JSON из файла (локальный .json).** Если у вас есть файл `config.json` в папке проекта (например, в `assets/` во Flutter), сначала читаем файл в строку, потом парсим:

```dart
import 'dart:convert';
import 'package:flutter/services.dart' show rootBundle;

// Flutter: файл в assets/config.json
Future<Map<String, dynamic>> loadJsonFromAssets() async {
  final String raw = await rootBundle.loadString('assets/config.json');
  return jsonDecode(raw) as Map<String, dynamic>;
}
```

Или с диска (Dart без Flutter или когда путь к файлу известен):

```dart
import 'dart:convert';
import 'dart:io';

Future<Map<String, dynamic>> loadJsonFromFile(String path) async {
  final file = File(path);
  final String raw = await file.readAsString();
  return jsonDecode(raw) as Map<String, dynamic>;
}
```

**Шаг 4: из Map в свой класс (fromJson).** Чтобы не работать с «голыми» ключами и типами `dynamic`, данные оборачивают в модель — класс с полями. В конструкторе или фабричном методе **fromJson** читаем ключи из `Map` и присваиваем полям. Обязательно обрабатываем отсутствующие ключи (`json['name'] ?? ''`) и разные типы (число может прийти как int или double, флаг — как bool или строка "true"):

```dart
class User {
  final String id;
  final String email;
  final String name;

  User({required this.id, required this.email, required this.name});

  factory User.fromJson(Map<String, dynamic> json) {
    return User(
      id: (json['id'] ?? '').toString(),
      email: (json['email'] ?? '').toString(),
      name: (json['name'] ?? '').toString(),
    );
  }
}

// использование:
final data = jsonDecode(response.body);
User user = User.fromJson(data);
print(user.name);
```

**Шаг 5: парсинг списка JSON.** Если сервер вернул массив объектов, после `jsonDecode` получаем `List<dynamic>`. Проходим по списку и каждый элемент приводим к `Map`, затем создаём объект модели:

```dart
final List<dynamic> rawList = jsonDecode(response.body);
List<User> users = [];
for (var item in rawList) {
  users.add(User.fromJson(item as Map<String, dynamic>));
}
// или короче:
List<User> users = (jsonDecode(response.body) as List)
    .map((e) => User.fromJson(e as Map<String, dynamic>))
    .toList();
```

**Обратная операция: объект в JSON (toJson).** Чтобы отправить данные на сервер или сохранить в файл, модель превращают обратно в Map, затем в строку:

```dart
Map<String, dynamic> toJson() => {
  'id': id,
  'email': email,
  'name': name,
};

String jsonString = jsonEncode(user.toJson());
```

В проекте чата так устроены модели `Chat`, `Message`, `User`: у каждой есть `fromJson` (и у части — `toJson`); ответы API парсятся через `jsonDecode(response.body)` в сервисах (`AuthService`, `ChatsService`, `MessagesService`).

### Классы и объекты

Когда мы описываем «что такое чат» или «что такое сообщение», мы перечисляем свойства: у чата есть имя, id, количество непрочитанных; у сообщения — текст, отправитель, дата. В программировании такой «чертёж» называется **класс**, а конкретный чат или конкретное сообщение — **объект** (экземпляр класса). Один класс — много объектов: один класс «Чат», а чатов в приложении много. В нашем проекте модели данных (Chat, Message, User) — это как раз классы с полями и способом создать объект из данных с сервера (из JSON).

В проекте модели данных — это классы с полями и фабричными конструкторами для разбора JSON.

**Пример — модель чата** в `lib/models/chat.dart`:

```dart
class Chat {
  final String id;
  final String name;
  final bool isGroup;
  final String? folderId;
  final String? folderName;
  final int unreadCount;
  // ... другие поля

  Chat({
    required this.id,

    this.folderId,
    this.folderName,
    this.unreadCount = 0,
    // ...
  });

  factory Chat.fromJson(Map<String, dynamic> json) {
    // создаёт Chat из JSON с сервера
    return Chat(
      id: (json['id'] ?? '').toString(),
      name: json['name'] ?? '',

```

**Модель сообщения** в `lib/models/message.dart` — класс `Message` с полями `id`, `chatId`, `content`, `senderEmail`, `createdAt` и т.д., плюс геттеры вроде `hasImage`, `status`, и методы `fromJson` / `toJson`.

### Null-safety: ?, !, ??

Иногда значения может не быть: например, у чата может не быть названия папки или у сообщения — картинки. В Dart такое «пустое» значение обозначается словом `null`. Чтобы программа не «падала» из-за неожиданного отсутствия данных, в Dart включена **null-safety**: по умолчанию переменная не может быть `null`, если мы явно не разрешим это. Для этого и нужны значки `?`, `!` и `??`:

- **Тип с `?`** — переменная может быть null: `String? name` значит «строка или ничего».
- **`!`** — мы уверены: «здесь уже точно не null» (если окажется null — будет ошибка при выполнении).
- **`??`** — «если слева null, возьми значение справа»: `data['username'] ?? 'гость'` — если имени нет, подставится «гость».

**Примеры из проекта:**

В `lib/main.dart` при построении темы используется `_appTheme!`, потому что к моменту использования тема уже гарантированно задана. В сервисах часто встречается получение токена и проверка на null:

```dart
final token = await StorageService.getToken();
if (token == null) throw Exception('Требуется авторизация');
```

И использование значения по умолчанию: `json['name'] ?? ''`, `prefs.getBool(key) ?? false`.

### Асинхронность: Future, async/await

Часть действий занимает время: запрос в интернет, чтение данных с диска. Если бы программа «замирала» и ждала ответа, приложение бы подвисало и пользователь не мог бы ничего нажимать. Поэтому такие операции делают **асинхронно**: программа отправляет запрос и продолжает работать; когда ответ приходит, выполняется следующий кусок кода. В Dart результат «действия в будущем» описывается типом **Future**; слова **async** и **await** позволяют писать такой код почти как обычный последовательный: «подожди результат этого запроса, затем сделай следующее».

**Примеры из проекта:**

- Вход пользователя в `lib/screens/login_screen.dart`:

```dart
void _login() async {
  setState(() {
    _isLoading = true;
    _errorMessage = null;
  });
  try {
    final userData = await _authService.loginUser(username, password);
    if (!mounted) return;
    setState(() {
      _isLoading = false;
    });
    // переход на следующий экран...
  } catch (e) {
    if (mounted) {
      setState(() {
        _isLoading = false;
        _errorMessage = e.toString().replaceFirst('Exception: ', '');
      });
    }
  }
}
```

- Загрузка чатов в `lib/screens/home_screen.dart`:

```dart
Future<void> _loadChats() async {
  setState(() {
    _isLoading = true;
    _loadError = null;
  });
  try {
    final chats = await _chatsService.fetchChats(widget.userId);
    final order = await StorageService.getChatOrder(widget.userId);
    if (mounted) {
      setState(() {
        _chats = chats;
        _chatOrder = order;
        _loadError = null;
      });
    }
  } catch (e) {
    if (mounted) {
      setState(() => _loadError = e.toString().replaceFirst('Exception: ', ''));
      ScaffoldMessenger.of(context).showSnackBar(/* ... */);
    }
  } finally {
    if (mounted) setState(() => _isLoading = false);
  }
}
```

Сервисы `AuthService`, `ChatsService`, `MessagesService` строятся на `Future`: методы возвращают `Future<...>`, внутри используется `await` для HTTP-запросов и работы с хранилищем.

### Обработка ошибок: try/catch

Сеть может быть недоступна, сервер может вернуть ошибку, данные — оказаться в неожиданном формате. Если не обработать такую ситуацию, приложение может «упасть». Поэтому опасный код оборачивают в блок **try**: «попробуй выполнить это». Если внутри произошла ошибка, выполнение переходит в блок **catch**: там мы показываем пользователю сообщение («Не удалось загрузить чаты»), предлагаем «Повторить» или просто логируем ошибку для отладки. Приложение остаётся рабочим, а пользователь понимает, что пошло не так.

В проекте типичный паттерн: в `try` — вызов сервиса (`loginUser`, `fetchChats`, `fetchMessages`), в `catch` — обновление состояния экрана (сообщение об ошибке, кнопка «Повторить») и опционально вывод в консоль в режиме отладки. Пример уже приведён выше для `_login` и `_loadChats`. В сервисах исключения пробрасываются выше (`rethrow`) или оборачиваются в `Exception` с понятным текстом для пользователя.

## Часть C. Flutter с нуля

В части B мы разбирали язык Dart: переменные, условия, функции, классы. Всё это нужно, чтобы обрабатывать данные и логику. Но пользователь видит не переменные, а **экран**: кнопки, текст, списки. За то, как это рисуется и как реагирует на нажатия, отвечает **Flutter**. В этой части мы разберём: что такое виджет, как устроено «дерево» элементов на экране, как экран обновляется при изменении данных (состояние) и как переходить с одного экрана на другой (навигация).

### Что такое Flutter и зачем Dart

**Простыми словами:** Flutter — это инструмент (фреймворк) для создания интерфейсов приложений. Один и тот же код можно собрать под телефон (iOS, Android) и под браузер (Web). Язык Flutter — **Dart**; компилятор переводит его в нужный формат для каждой платформы.

**Теория:** интерфейс в Flutter описывается декларативно — вы говорите, как экран должен выглядеть в зависимости от данных; при изменении данных Flutter сам перерисовывает только нужные части.

### Виджеты: StatelessWidget и StatefulWidget

**Виджет (widget)** — элемент интерфейса: кнопка, текст, экран, список. Всё в Flutter — виджет. Дерево виджетов образует весь экран.

- **StatelessWidget** — виджет без внутреннего изменяемого состояния. Ему передают параметры (например, цвет схемы, текст), он только строит интерфейс по ним. Пример в проекте — `EulaConsentScreen` в `lib/screens/eula_consent_screen.dart`: экран с фиксированным текстом и кнопкой «Принимаю».
- **StatefulWidget** — виджет с состоянием (данные, введённый текст, флаг загрузки). Состояние хранится в отдельном объекте `State`; при вызове `setState()` Flutter перестраивает виджет с новыми данными.

**Примеры из проекта:**

- `LoginScreen` в `lib/screens/login_screen.dart` — `StatefulWidget`: есть контроллеры полей ввода, `_errorMessage`, `_isLoading`; при нажатии «Войти» вызывается `_login()`, внутри `setState` обновляет индикатор и сообщение об ошибке.
- `ChatInputBar` в `lib/widgets/chat_input_bar.dart` — `StatelessWidget`: получает `controller`, `onSend`, `scheme` и т.д. и только отображает поле ввода и кнопки; состояние текста живёт в переданном `TextEditingController` снаружи.

### Дерево виджетов и BuildContext

Виджеты образуют дерево: корень — приложение (`MaterialApp`), ниже — экраны, затем контейнеры, кнопки, текст. **BuildContext** — объект, который передаётся в метод `build` и даёт доступ к теме, навигатору, медиа-запросу. Например, `Theme.of(context).colorScheme`, `Navigator.push(context, ...)`, `ScaffoldMessenger.of(context).showSnackBar(...)`.

Flutter перерисовывает экран, когда меняется состояние (после `setState`) или при первой отрисовке. Перестраиваются только те части дерева, чьи данные изменились.

### Состояние: setState

**Простыми словами:** состояние — это данные, которые могут меняться и от которых зависит, что показать на экране (список чатов, текст ошибки, индикатор загрузки). Когда вы меняете эти данные, нужно сообщить Flutter о перерисовке — для этого вызывают `setState(() { ... })`, внутри обновляют поля состояния.

В проекте примеры: форма входа (`_isLoading`, `_errorMessage`), список чатов (`_chats`, `_loadError`, `_isLoading`), поле ввода сообщения (состояние хранится в `TextEditingController`, который передаётся в `ChatInputBar`). После асинхронной операции перед вызовом `setState` часто проверяют `if (!mounted) return;`, чтобы не обновлять уже размонтированный виджет.

### Тема и стили: ThemeData, AppColors

Единый вид приложения задаётся темой. В проекте тема строится в `lib/main.dart` в методе `_buildTheme()`: используется `ColorScheme.dark` с цветами из `AppColors` (`lib/theme/app_colors.dart`). Задаются цвета фона, карточек, кнопок, полей ввода, диалогов, SnackBar и т.д. Виджеты берут цвета через `Theme.of(context).colorScheme` или напрямую `AppColors.primary`, `AppColors.primaryGlow` и т.д.

**Пример** — фрагмент из `lib/main.dart`:

```dart
const scheme = ColorScheme.dark(
  primary: AppColors.primary,
  onPrimary: AppColors.onSurfaceDark,
  surface: AppColors.surfaceDark,
  onSurface: AppColors.onSurfaceDark,
  // ...
);
return ThemeData(
  brightness: Brightness.dark,
  colorScheme: scheme,
  scaffoldBackgroundColor: surface,
  // ...
);
```

### Навигация: Navigator, push/pop

**Простыми словами:** навигация — переход с одного экрана на другой. `Navigator.push` — открыть новый экран поверх текущего; `Navigator.pop` — закрыть текущий и вернуться к предыдущему. При `push` можно передать аргументы через конструктор экрана.

**Примеры из проекта:**

- Переход с логина на главный экран или на EULA в `lib/screens/login_screen.dart`:

```dart
Navigator.pushReplacement(

```

- Открытие экрана чата из списка чатов в `lib/screens/home_screen.dart`:

```dart
void _openChat(Chat chat) {
```

### Работа с сетью: HTTP и сервисы

Приложение общается с API по HTTP. В проекте используется пакет `http`. Базовый URL задаётся в `lib/config/api_config.dart` (можно переопределить через `--dart-define=API_BASE_URL=...`). Токен хранится в `StorageService` (на мобильных — в secure storage, на web — в SharedPreferences) и подставляется в заголовок `Authorization: Bearer <token>` в методах вида `_getAuthHeaders()` в `AuthService`, `ChatsService`, `MessagesService`.

**Пример** — запрос списка чатов в `lib/services/chats_service.dart`:

```dart
Future<List<Chat>> fetchChats(String userId) async {
  final headers = await _getAuthHeaders();
  final response = await http.get(
    Uri.parse('$baseUrl/chats'),
    headers: headers,
  );
  if (response.statusCode == 200) {
    final List<dynamic> data = jsonDecode(response.body);
    return data.map((json) => Chat.fromJson(json as Map<String, dynamic>)).toList();
  } else {
    throw Exception('Не удалось загрузить чаты: ${response.statusCode}');
  }
}
```

Аналогично устроены логин, отправка сообщений, загрузка сообщений по чату и т.д.

### Реальное время: WebSocket

Чтобы новые сообщения появлялись без перезагрузки, в проекте используется **WebSocket**: постоянное соединение с сервером. Сервис `WebSocketService` в `lib/services/websocket_service.dart` — синглтон (`WebSocketService.instance`). Он подключается с токеном, получает события (например, новое сообщение) и отдаёт их через `stream`. В `HomeScreen` и на экране чата подписываются на этот поток и при получении события обновляют список чатов или список сообщений.

**Пример подписки** в `lib/screens/home_screen.dart`:

```dart
void _subscribeToNewMessages() {
  WebSocketService.instance.connectIfNeeded();
  _wsSubscription?.cancel();
  _wsSubscription = WebSocketService.instance.stream.listen((event) {
    if (!mounted) return;
    if (event is Map && event['chat_id'] != null && event['id'] != null) {
      _loadChats();
    }
  });
}
```

В `dispose()` подписка отменяется: `_wsSubscription?.cancel();`.

## Часть D. Проект как целое

### Точка входа: main.dart

При запуске приложения выполняется функция `main()`. В проекте в `lib/main.dart`:

1. Настраивается глобальная обработка ошибок Flutter и асинхронных ошибок (`runZonedGuarded`).
2. Инициализируются биндинги Flutter, форматирование дат (`intl`), локальное хранилище сообщений, push-уведомления.
3. Запускается приложение `runApp(const MyApp())`.

Виджет `MyApp` — корень дерева. В `build` возвращается `MaterialApp` с темой и с `home`, заданным через `FutureBuilder`: асинхронно загружаются данные пользователя и флаг принятия EULA из `StorageService`. В зависимости от результата показывается экран загрузки, `LoginScreen`, `EulaConsentScreen` или `MainTabsScreen` (главный экран с вкладками и чатами). Таким образом, при наличии сохранённого токена пользователь сразу попадает на нужный экран без повторного ввода логина.

### Маршруты и экраны

Типичный поток: **Логин** → (при первом входе) **EULA** → **Главный экран с чатами** (вкладки: чаты и др.) → по тапу на чат — **Экран чата**. Из главного экрана доступны **Профиль**, смена пароля, выход. Данные между экранами передаются через параметры конструктора (например, `userId`, `userEmail`, `chatId`, `chatName`). Глобальный ключ навигатора `navigatorKey` используется для перехода к экрану из контекста push-уведомлений.

### Сервисы: кто за что отвечает

| Сервис | Назначение |

### Модели данных: JSON → объекты Dart

Сервер отдаёт данные в формате JSON (текст в виде массивов и объектов). В проекте каждая сущность представлена классом с фабричным конструктором `fromJson` и при необходимости методом `toJson`:

- **Chat** — `lib/models/chat.dart`: поля `id`, `name`, `isGroup`, `lastMessageText`, `lastMessageAt`, `unreadCount` и др.; разбор из `json['id']`, `json['last_message']` и т.д.
- **Message** — `lib/models/message.dart`: поля `id`, `chatId`, `content`, `senderEmail`, `createdAt`, вложения, реакции, статус прочтения; вложенные объекты (например, `reply_to_message`) парсятся рекурсивно через `Message.fromJson`.
- **User** — `lib/models/user.dart`: минимальная модель `id`, `email`.

Так данные с сервера превращаются в типизированные объекты Dart, с которыми удобно работать в коде.

### Типичные задачи пошагово

- **Добавить поле на экран:** найти соответствующий экран в `lib/screens/`, добавить в состояние (если нужно) переменную или использовать переданные параметры, в `build` добавить виджет (например, текст, поле ввода). При необходимости вызвать сервис и в `setState` обновить состояние.
- **Изменить текст кнопки/подпись:** открыть нужный экран или виджет, найти строку с текстом (например, `'Войти'`, `'Отправить'`) и изменить её.
- **Подключить новый API-метод:** в соответствующем сервисе (`AuthService`, `ChatsService`, `MessagesService`) добавить метод с `async`/`await`, вызвать `http.get`/`post` с URL и заголовками из `_getAuthHeaders()`, распарсить ответ (JSON → модель или Map). На экране вызвать этот метод и обработать результат (обновить состояние, показать ошибку через SnackBar).

## Часть E. Дополнительно

### Как запустить проект локально

1. Установите Flutter SDK и добавьте его в PATH.
2. В корне проекта выполните:
   - `flutter pub get` — установка зависимостей.
   - `flutter run` — запуск на подключённом устройстве или эмуляторе (по умолчанию выберется платформа).
3. Для web: `flutter run -d chrome` (или другой браузер).
4. Для сборки под платформу: `flutter build web`, `flutter build apk`, `flutter build ios` (среды и подписи настраиваются отдельно).

Базовый URL API задаётся в `lib/config/api_config.dart`; при необходимости переопределите его при сборке:  
`flutter run --dart-define=API_BASE_URL=https://ваш-сервер`.

### Где искать ошибки

- **Консоль** (терминал, где запущен `flutter run`): вывод `print`, трассировки необработанных исключений.
- В коде в режиме отладки часто используется `if (kDebugMode) { print('...'); }` — такие сообщения не попадут в релиз.
- Типичные сообщения Flutter: «setState() called after dispose()» — вызов `setState` после того, как виджет уже снят с дерева (решение: проверять `mounted` перед `setState`). Ошибки сети и парсинга JSON обычно видны в catch-блоках сервисов и на экране в виде SnackBar или текста ошибки.

## Словарик терминов

| Термин | Краткое определение |
|--------|----------------------|
| **Dart** | Язык программирования, на котором пишется код Flutter-приложения. |
| **Тип** | Описание рода данных: `int`, `String`, `bool`, `List`, `Map` и т.д. |
| **Null-safety** | Система типов Dart, в которой переменные по умолчанию не могут быть null; nullable обозначается `?`. |
| **Future** | Обещание результата асинхронной операции; тип возвращаемого значения функций с `async`. |
| **async/await** | Синтаксис для асинхронного кода: `await` ждёт завершения `Future`, не блокируя поток. |
| **Виджет (widget)** | Элемент интерфейса во Flutter: кнопка, текст, экран, список — всё строится из виджетов. |
| **StatelessWidget** | Виджет без собственного изменяемого состояния; только параметры и build. |
| **StatefulWidget** | Виджет с состоянием; состояние хранится в объекте State, обновляется через setState. |
| **State** | Класс, хранящий изменяемые данные виджета и метод build для отрисовки. |
| **setState** | Метод State: помечает виджет как «грязный» и запускает перерисовку с новыми данными. |
| **BuildContext** | Контекст сборки виджета; даёт доступ к теме, навигатору, медиа-запросу. |
| **MaterialApp** | Корневой виджет приложения Material Design: тема, маршруты, локаль. |
| **Navigator** | Управление стеком экранов: push (открыть), pop (закрыть). |
| **Сервис** | Класс с логикой, не связанной напрямую с UI: сеть, хранилище, WebSocket. |
| **Модель** | Класс, описывающий структуру данных (например, Chat, Message); часто с fromJson/toJson. |
| **HTTP** | Протокол запросов к API: GET, POST и т.д.; в проекте через пакет `http`. |
| **WebSocket** | Постоянное двустороннее соединение для событий в реальном времени (новые сообщения). |
| **JSON** | Текстовый формат обмена данными; в Dart парсится через `jsonDecode`, собирается через `jsonEncode`. |
| **Токен** | Строка авторизации после входа; передаётся в заголовке `Authorization: Bearer <token>`. |
| **Код** | Текст программных инструкций в файлах. |
| **Файл** | Один «документ» на диске с кодом или данными (например, `main.dart`). |
| **Папка** | Каталог, в котором лежат файлы и другие папки (например, `lib/screens/`). |
| **Сервер** | Программа в интернете, которая принимает запросы и отдаёт данные (чаты, сообщения). |
| **Клиент** | Приложение на устройстве пользователя (телефон, браузер), которое рисует интерфейс и шлёт запросы серверу. |
