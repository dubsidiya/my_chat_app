# Бэклог идей — что можно сделать в приложении

Файл-черновик с идеями по развитию **reollity / my_chat_app** после серии задач по фото-вьюверу, композеру и видео.

Идеи сгруппированы по областям. Помечены ориентировочной приоритетностью (P0 — высокий, P1 — средний, P2 — низкий) и масштабом работ (S/M/L).

> Это идеи, а не план. Перед реализацией — уточнять область, ставить отдельный план, прогонять `flutter analyze` и серверные smoke-проверки согласно `.cursor/rules/project-workflow.mdc`.

> **Статус (2026-06):** ✅ — уже в приложении; ⏳ — частично. Сводка реализованного — в `FEATURES_IDEAS.md` и `FEATURE_RECOMMENDATIONS.md`.

---

## 1. Медиа в чате (продолжение телеграм-лайт линии)

- **P0 / M** — Единый media-viewer для фото и видео (свайп между ними, как в Telegram). Сейчас фото в `ChatFullscreenImageViewer`, видео — в `VideoPlayerScreen`. Объединить.
- **P0 / S** — Превью видео прямо в композере перед отправкой (сейчас выбранное видео показывается как «файл» с иконкой). Использовать `VideoThumbnailView`.
- **P0 / S** — Прогресс загрузки видео/файла в bubble (сейчас прогресс глобальный, не на самой карточке).
- **P1 / M** — Кеш видео-файлов локально (`flutter_cache_manager`), чтобы повторно открывалось мгновенно.
- **P1 / S** — Длительность и preview ещё до тапа: бэкенд может сохранять `video_duration_ms` и `video_thumb_url` при upload (через ffprobe или клиент-сайд при отправке).
- **P1 / M** — Сжатие/перекодировка видео перед отправкой (target ~720p, h264 baseline), чтобы экономить трафик и место.
- **P2 / M** — Picture-in-Picture (`floating`) для видео плеера на iOS/Android.
- **P2 / S** — Поддержка GIF-анимаций как отдельного типа (сейчас GIF идёт как картинка).
- **P2 / M** — Альбомы из нескольких фото/видео в одном сообщении.

## 2. Композер сообщений

- **P0 / S** — Отдельный preview-блок для выбранного видео (сейчас показывается как файл).
- **P0 / S** — Отмена аплоада: x-кнопка на превью загрузки.
- **P1 / S** — Markdown / простая разметка (жирный, курсив, ссылки) через тулбар над клавиатурой.
- **P1 / M** — Эмодзи-пикер прямо в композере (нативный пикер мало подходит на десктопе).
- **P1 / M** — Стикеры/реакции наборами (как в Telegram). На бэкенде — таблица `stickers`, на клиенте — отдельный пикер.
- **P1 / S** — Drafts на чат: сохранять незаконченный текст между сессиями (Hive).
- **P2 / S** — Авто-определение видео-ссылок YouTube/Vimeo и встроенный плеер (на основе link preview).

## 3. Сообщения и общение

- **P0 / M** — Forward to multiple chats (мультивыбор чатов при пересылке). (Пересылка в один чат ✅.)
- **P0 / M** — Edit history popup для отредактированных сообщений.
- **P1 / M** — Цитирование куска текста (не всего сообщения целиком).
- **P1 / M** — Pinned chats и `unread mentions`-фильтр в списке чатов. (Закреплённые **сообщения** ✅; pinned chats — нет.)
- **P1 / L** — Threads (треды от конкретного сообщения, как в Slack).
- **P1 / S** — «Печатает: имя…» в шапке для группового чата с несколькими печатающими.
- **P2 / M** — Schedule send (отправить позже).
- **P2 / M** — Self-destruct messages (по таймеру).

## 5. Поиск и навигация

- **P1 / M** — Глобальный поиск по сообщениям всех чатов (бэкенд: full-text index в Postgres, `tsvector`).
- **P1 / M** — Фильтры в поиске: только медиа, только файлы, только ссылки, по дате, по отправителю.
- **P1 / S** — Прыжок к дате (date jumper) в чате.
- **P2 / S** — Сохранённые сообщения (отдельный «чат с самим собой»).

## 6. E2EE и безопасность

> ⏳ Базовый E2EE для чатов уже есть (`E2eeService`); ниже — доработки.

- **P0 / M** — Доделать E2EE для аватаров и для текста в push-уведомлениях (сейчас в push прилетают clear-метаданные — см. `E2EE_METADATA_THREAT_MODEL.md`).
- **P0 / S** — Индикатор статуса E2EE в шапке чата с возможностью посмотреть fingerprint собеседника.
- **P1 / M** — Rotation ключей чата по событию (выход участника, через N дней).
- **P1 / M** — Hardware-backed key storage (Android Keystore / iOS Secure Enclave) для приватного X25519-ключа.
- **P2 / M** — Disappearing keys для secret-чатов.

## 10. Производительность и качество

- **P0 / S** — Жёсткий лимит размера `E2eeImage` кеша (сейчас Future кеш в state, но без LRU).
- **P0 / S** — Lazy initialization `VideoPlayerController` в `VideoThumbnailView` только если duration ещё не известен (сейчас всегда + thumbnail).
- **P0 / M** — Изолировать тяжёлые операции (сжатие изображений, шифрование медиа) в `Isolate` через `compute`.
- **P1 / M** — Прогрессивная загрузка истории: загружать сначала текстовые сообщения, медиа догружать позже.
- **P1 / S** — Skeleton placeholders для медиа-сетки.
- **P2 / M** — Сегментирование больших списков через `slivers` + `SliverList.builder` с keep-alive только для последних N сообщений.

## 12. Бэкенд

- **P0 / S** — Видео-thumbnails на стороне сервера (ffmpeg) и кладём `file_thumb_url`. Это уберёт двойную сетевую загрузку на клиенте.

- **P2 / M** — gRPC/HTTP/2 для медиа-загрузок (стримминг чанков).

## Как пользоваться этим файлом

- Каждая идея — самостоятельная единица.
- Перед взятием идеи в работу: открыть отдельный план в Cursor (Plan mode), выяснить scope и побить на под-задачи.
- Если идея затрагивает `accounting/reports`-логику — обязательно добавлять smoke-сценарий (`npm run smoke:reports:permissions` и т.п.).
- Не забывать про правила доступа к отчётам и про E2EE-обещания.
