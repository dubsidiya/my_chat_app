# Исправление ошибки сборки Flutter на Vercel

## Проблема
```
Build Failed
Command "flutter pub get" exited with 127
```

Ошибка 127 означает, что команда `flutter` не найдена. Vercel по умолчанию не имеет Flutter установленного.

## Решение

Я создал скрипт `build.sh`, который:
1. Устанавливает Flutter перед сборкой
2. Устанавливает зависимости
3. Собирает веб-версию

## Что было сделано:

1. ✅ Создан `build.sh` - скрипт для установки Flutter и сборки
2. ✅ Обновлен `vercel.json` - теперь использует `build.sh` вместо прямых команд

## Альтернативное решение (если build.sh не работает)

Если скрипт не работает, можно использовать GitHub Actions для сборки:

1. Создайте `.github/workflows/build.yml`
2. Настройте автоматическую сборку при каждом коммите
3. Загружайте только папку `build/web` в Vercel

Или используйте готовую сборку:
- Соберите локально: `flutter build web`
- Закоммитьте папку `build/web`
- Vercel будет использовать готовую сборку

## Проверка

После коммита изменений:
1. Vercel запустит `build.sh`
2. Скрипт установит Flutter
3. Выполнит сборку
4. Результат будет в `build/web`

Если все еще не работает, проверьте логи сборки в Vercel Dashboard.
